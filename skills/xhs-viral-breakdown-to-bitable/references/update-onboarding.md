# 更新用户引导

本页只用于“已有旧版配置”的更新流程，不用于首次安装。

## 必须执行

1. 运行 `upgrade-check`，只读检查当前绑定表的字段和默认视图。
2. 若返回 `mode: upgrade`，把 `customer_message` 原样发给用户。消息已经包含：本次更新内容、原表需要怎样调整、准备如何执行，以及授权问题。
3. 停止后续操作，等待用户明确同意。沉默、继续使用 Skill 或之前对其他操作的同意，都不算本次授权。
4. 用户同意后，使用本次返回的 `plan_id` 执行：

   ```text
   node "<skill-dir>/scripts/xhs-breakdown.mjs" upgrade-apply --plan-id <plan_id> --confirm-upgrade --output-dir <报告目录>
   ```

5. 若提示方案变化，重新运行 `upgrade-check`、重新展示说明并重新授权。

升级只新增缺失字段、增量调整默认视图并保存前后报告，不删除旧字段、旧数据或自定义字段。升级完成后可以另行询问是否预览历史图片修复，但不能直接执行；表结构授权不包含历史修复授权。
