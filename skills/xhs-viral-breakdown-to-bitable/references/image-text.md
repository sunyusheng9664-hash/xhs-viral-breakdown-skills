# 图文分析与字段

只处理 `type=image_text` 的记录。

## analysis 对象

```json
{
  "body_summary": "正文结构化摘要",
  "cover_analysis": "封面分析",
  "interaction_drivers": "互动诱因",
  "viral_reasons": "爆款原因",
  "reusable_tactics": "可复制点"
}
```

## 分析要求

- `body_summary`：概括主题、痛点、展开方式和承诺结果，不复制完整原文；补充开头方式、中段逻辑、结尾设计与信息密度。
- `cover_analysis`：使用 VISUAL-D：视觉焦点（Viewport）、信息层级（Information）、信号类型（Signal）、差异化（Uniqueness）、行动暗示（Action）、排版/视觉动线（Layout）、决策成本（Decision）。每个已观察到的维度都要给出具体证据，而不是“精美”“吸引人”等形容词。
- `interaction_drivers`：使用 TRIGGER：时机、奖励、身份、信息缺口、收益、自我表达、风险；区分主要与次要驱动力，并分别说明评论、收藏、转发的触发机制。
- `viral_reasons`：使用三层归因：必要条件（需求、稀缺性、真实性）、充分条件（封面决策成本、3 秒价值交付、互动机制、搜索/推荐适配）、放大器（时机、作者、平台、社交货币）。明确哪些能复制、哪些只是外部放大因素。
- `reusable_tactics`：必须给出可替换变量的公式：标题、封面、正文、互动设计和信任建立；每一项都写明适用条件，不能只给“标题用数字法”一类描述。

如果没有实际检查封面或页面，`cover_analysis` 必须说明分析仅基于标题、正文和封面 URL；不得虚构颜色、排版、画面元素、评论区置顶或同类内容对比。无法判断的维度直接标注“未观察到”。

## 输出骨架

```text
封面分析（VISUAL-D）
- 视觉焦点：……（证据：……）
- 信息层级：……（证据：……）
- 信号/差异化/行动暗示/排版/决策成本：……（证据或未观察到）
- 综合：通过【机制】降低【用户的理解或决策成本】。

互动诱因（TRIGGER）
- 主要驱动力：……；次要驱动力：……
- 评论/收藏/转发分别因……而发生。

爆款原因（三层归因）
- 必要条件：……
- 充分条件：……
- 放大器条件：……
- 因果链：……

可复制点（公式化）
- 标题：【变量】+【变量】+【变量】；适用条件：……
- 封面：【变量】+【变量】+【变量】；适用条件：……
- 正文：…… → …… → …… → CTA；变量替换：……
- 互动/信任：在【位置】以【方式】引导【动作】；适用条件：……
```

## 质量门槛

- 每个判断都能追溯到标题、正文、封面、字幕或公开元数据中的证据。
- 公式必须可直接替换变量复用，不能复述原笔记。
- 不把单条内容的结果归因为平台扶持、粉丝量或流量倍数，除非公开数据已证实。

## 飞书字段顺序

```text
标题
正文
链接
话题
封面
点赞
收藏
评论
转发
图片
封面分析
互动诱因
爆款原因
可复制点
```

其中点赞、收藏、评论、转发为数字；图片为封面之后的图片 URL，以换行分隔。
