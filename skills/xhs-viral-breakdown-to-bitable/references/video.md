# 视频分析与字段

只处理 `type=video` 的记录。

## analysis 对象

```json
{
  "viral_breakdown": "爆款拆解",
  "reusable_tactics": "可复制部分"
}
```

## 分析要求

`viral_breakdown` 使用四维诊断：时间轴、信息密度、信任建立、转化设计；`reusable_tactics` 抽象成可独立复用的模块和完整脚本公式，不照搬来源内容。

- 时间轴：按 0–3 秒钩子、3–10 秒问题建立、10–30 秒价值交付、30 秒至结束的收尾/CTA 分段；有时间戳则标注时间，无时间戳只写相对阶段，不能伪造精确秒数。
- 信息密度：识别高密度段、低密度段及转折点，说明其功能；仅有字幕时，以文本长度和信息点频率判断。
- 信任建立：区分真实感（具体经历/不完美表达）、专业性（数据/案例/方法）、共情力（对痛点与身份的理解），并说明其可复制前提。
- 转化设计：识别 CTA 的类型、位置、时机、行动门槛和即时回报；无 CTA 时明确写“未观察到明确 CTA”。
- 可复制模块：每个模块写明功能、结构、可替换变量、适用场景、建议时长/节奏。最后给出“钩子 → 问题建立 → 价值交付 → 信任强化 → CTA”的可替换脚本公式。

只有字幕和元数据时，先写“【分析限制说明】本分析仅基于字幕文本和公开元数据，未检视视频画面。”；使用“字幕显示”“从口播结构看”“适合配合”等谨慎表述。未检查视频帧时，不得写“画面展示了”，也不得把推断写成事实。

字幕经过高置信度纠错后写入历史字段 `口播文案（原始字幕）`。常见纠错包括：

```text
codes/codax/codesi -> Codex
s q/sq -> Skill
华人勋 -> 黄仁勋
mac档 -> Markdown
卡巴西 -> 卡帕西
入手 -> 助手
外科手术师修改 -> 外科手术式修改
```

不确定的词保留原文，不擅自润色说话风格。无字幕时写 `未获取字幕`，继续分析可确认的元数据。

## 输出骨架与质量门槛

```text
爆款拆解（四维诊断）
- 时间轴：0–3 秒【钩子/机制】；问题建立【功能】；价值交付【信息】；收尾【CTA 或未观察到】。
- 信息密度：高密度段……；低密度段……；变化原因……。
- 信任建立：真实感/专业性/共情力分别由……支撑；主要依赖……，其复用前提是……。
- 转化设计：CTA 类型、位置、时机、门槛、即时回报……。
- 综合机制：用【钩子】解决“为什么看”，用【结构】解决“为什么继续看”，用【信任】解决“信不信”，用【设计】解决“动不动”。

可复制部分
- 模块：【名称】；功能：……；结构：……；变量：……；适用场景：……；节奏：……。
- 脚本公式：【钩子】+【问题建立】+【价值交付】+【信任强化】+【CTA】。
```

- 每个判断必须有字幕、公开元数据或已检查画面的证据。
- 不使用“节奏好”“有感染力”“内容优质”等无证据判断。
- 不估算数据提升比例；无法判断的部分标注“未观察到”或“无法确认”。

## 飞书字段顺序

```text
标题
链接
封面
作者
视频时长
点赞数
收藏数
评论数
分享数
正文
口播文案（原始字幕）
爆款拆解
可复制部分
```

点赞数、收藏数、评论数、分享数为数字。
