# Image-Text Analysis Templates
Use these as structure and diagnostic framework, not as boilerplate text.
Each analysis must be specific, evidence-based, and operational.

## 正文
```text
这篇笔记围绕【主题】展开，先指出【痛点/反常识/冲突】，再给出【方法/步骤/案例/工具/清单】，最终承诺【结果/价值/改变】。核心判断是：【一句话提炼——这个笔记的本质是在卖什么：希望/恐惧/身份认同/实用工具/情绪共鸣】。

内容结构诊断：
- 开头方式：【直接给结果/制造悬念/指出反常识/引发共鸣】
- 中段逻辑：【并列清单/时间顺序/问题-解决/对比论证/步骤教程】
- 结尾设计：【CTA引导/情绪升华/下一篇预告/开放性问题】
- 信息密度：【高（每屏都有新信息）/中（适当留白）/低（以情绪/氛围为主）】
```

## 封面分析
使用 VISUAL-D 诊断模型，逐项检查并给出具体证据：

```text
V - Viewport（视觉焦点）：用户第一眼看到的是什么？
  证据：【具体描述画面中最突出的元素，如"左上角的红色大字'30天'占封面30%面积"】
  
I - Information（信息层级）：标题、数字、关键词的层级关系
  证据：【描述阅读顺序，如"第一层：数字'30'；第二层：痛点词'失业'；第三层：结果'月入3W'"】
  
S - Signal（信号类型）：封面传递的核心信号
  判断：【权威信号（专家/数据/认证）/ 利益信号（赚钱/省钱/省时）/ 情绪信号（焦虑/希望/愤怒）/ 社交信号（身份/品味/圈层）】
  证据：【具体元素如何传递该信号】
  
U - Uniqueness（差异化）：与同类笔记封面的差异点
  证据：【同类封面通常用X，这个封面用了Y，差异在于Z】
  
A - Action（行动暗示）：是否暗示了某种可执行的动作
  证据：【如"'跟着做'暗示用户需要行动，'收藏'暗示后续会用到"】
  
L - Layout（排版结构）：对齐方式、留白、视觉动线
  证据：【如"左对齐文字+右侧留白放截图，视觉动线从左到右"】
  
D - Decision（决策成本）：用户判断"要不要点"的成本高低
  判断：【低（一眼就知道能得到什么）/ 中（需要看标题辅助理解）/ 高（封面信息模糊）】
  证据：【为什么是这个判断】

综合结论：这个封面通过【核心机制】降低了用户的【什么成本】，从而提升了点击率。
```

**禁止输出**："封面精美""设计好看""标题吸引人"等无证据的形容词。
**必须输出**：每个维度至少一个具体证据。

## 互动诱因
使用 TRIGGER 诊断模型，识别主要驱动力和次要驱动力：

```text
T - Timing（时机）：互动入口设置在用户心理曲线的哪个节点？
  分析：【互动设计出现在用户情绪的哪个阶段——好奇期/共鸣期/收获期/行动期】
  
R - Reward（奖励）：用户互动后能获得什么？
  评论奖励：【获得解答/获得认同/表达观点/寻找同类】
  收藏奖励：【未来可复用的信息/省时工具/决策参考】
  转发奖励：【展示品味/传递价值/社交货币】
  
I - Identity（身份）：内容是否让用户产生代入感？
  分析：【是否使用了"你""我们""打工人"等身份标签，让用户觉得"这说的就是我"】
  
G - Gap（缺口）：是否制造了信息缺口？
  分析：【标题或正文中是否留下未完全解答的问题，迫使用户评论追问】
  
G - Gain（收益）：用户收藏后能否在未来获得明确收益？
  分析：【收藏动机是即时使用还是备用，收益是否具体可描述】
  
E - Ego（自我）：互动是否帮助用户表达自我？
  分析：【评论是否让用户展示知识/品味/态度/经历】
  
R - Risk（风险）：用户互动的心理门槛高低？
  分析：【评论是否可能暴露 ignorance/错误/不受欢迎的观点】

主要互动驱动力：【TRIGGER 中权重最高的 1-2 个维度】
次要互动驱动力：【权重次高的 1-2 个维度】

具体互动入口设计：
- 标题中埋入的钩子：【如"90%的人都做错了"——引发反驳欲】
- 正文中埋入的钩子：【如"评论区告诉我你的答案"——直接 CTA】
- 评论区置顶引导：【如有，描述具体引导方式】
```

## 爆款原因
使用三层归因模型，区分可复制的关键要素和外部因素：

```text
第一层：必要性条件（没有这些，一定不会爆）
- 选题需求强度：【高（刚需痛点）/中（兴趣需求）/低（可有可无）】
  证据：【这个痛点在用户日常中出现的频率，以及不解决的代价】
- 信息稀缺性：【独家信息/独特视角/旧信息新组合/常见信息但结构好】
  证据：【同类内容数量，以及本案例的差异化信息】
- 内容真实性：【真实案例/真实数据/真实体验/无法验证】
  证据：【具体使用了什么信任信号】

第二层：充分性条件（有了这些，爆的概率大幅提升——这是可复制部分）
- 封面决策成本：【低/中/高】，具体机制：【】
- 正文3秒法则：【开头是否在前3秒/前30字内交付核心价值或制造悬念】
- 互动动机设计：【是否在某处设计了"不得不互动"的节点】
- 算法友好度：
  * 搜索关键词匹配：【标题/正文是否包含高搜索量关键词】
  * 推荐流适配：【内容是否适合3秒内快速判断价值】
  * 互动数据反馈：【互动设计是否能在短时间内产生数据】

第三层：放大器条件（让爆款更爆——通常不可复制，只能借势）
- 时机因素：【是否踩中趋势/热点/节气/事件】
- 作者因素：【粉丝基数/专业背书/过往爆款积累】
- 平台因素：【是否获得官方流量扶持/话题页推荐】
- 社交货币：【内容是否具备"值得分享"的属性】

因果链条总结：
这个笔记的爆款路径是：【第一层要素】解决了"为什么值得看"，【第二层要素】解决了"为什么现在点"和"为什么互动"，【第三层要素】将结果放大了X倍。
```

## 可复制点
必须提炼为**公式化模板**，包含可替换变量和适用条件：

```text
1. 标题公式
结构：【公式，如"[数字] + [反常识判断] + [结果承诺]"】
示例：【原案例标题】→ 抽象后："30岁不是终点，而是月入3W的起点"
变量替换指南：【数字→时间/金额/数量；反常识判断→常见误解；结果承诺→具体收益】
适用条件：【什么类型的选题适合这个公式】

2. 封面公式
结构：【公式，如"[大字数字] + [痛点关键词] + [真实截图/数据凭证]"】
视觉规范：【配色建议/字体层级/元素位置】
适用条件：【什么类型的内容适合这个封面结构】

3. 正文结构公式
结构：【公式，如"痛点共鸣(20%) → 反常识观点(30%) → 方法论步骤(40%) → CTA(10%)"】
每段功能：【第一段解决___，第二段解决___...】
信息密度控制：【哪里快、哪里慢、哪里需要留白】

4. 互动设计公式
评论引导：【在___位置，用___方式，引导用户___】
收藏引导：【在___位置，暗示收藏后的___收益】
转发引导：【内容中是否包含___属性，让用户愿意分享】

5. 信任建立公式
第一层：【用什么建立真实感——具体细节/不完美/口语化】
第二层：【用什么建立专业性——数据/案例/方法论】
第三层：【用什么建立共情——理解痛点/表达情绪】
```

**要求**：每个公式必须能被用户直接套用，替换变量后即可生成新内容。
**禁止**：只给出描述性总结（如"标题用数字法"），必须给出可替换的公式结构。

## 分析质量检查清单
在输出最终分析前，确认以下检查项：
- [ ] 封面分析中是否每个 VISUAL-D 维度都有具体证据？
- [ ] 互动诱因中是否区分了主要驱动力和次要驱动力？
- [ ] 爆款原因中是否明确区分了三层归因？
- [ ] 可复制点中是否每个都是公式化模板（含可替换变量）？
- [ ] 全篇是否没有出现"吸引人""精美""有价值"等无证据形容词？
- [ ] 如果某些维度无法判断（如未看到评论区置顶），是否明确标注"未观察到"？
