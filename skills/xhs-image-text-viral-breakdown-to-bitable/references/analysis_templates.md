# Image-Text Analysis Templates

Use these as structure, not as boilerplate text.

## 正文

```text
这篇笔记围绕【主题】展开，先指出【痛点/反常识】，再给出【方法/步骤/案例/工具】，最终承诺【结果/价值】。核心判断是：【一句话提炼】。
```

## 封面分析

```text
封面使用【视觉类型】，主标题突出【关键词/数字/痛点】，辅助元素是【截图/表情包/流程图/清单/真实场景】。点击诱因在于【结果可见/痛点强/反常识/收藏价值】。
```

## 互动诱因

```text
互动主要来自【人群痛点】和【评论入口】。用户会因为【求工具/问方法/职业焦虑/对号入座/争议判断】评论；会因为【步骤清晰/模板可复用/清单完整/工具可尝试】收藏或转发。
```

## 爆款原因

```text
爆款成立原因是【选题热度】+【封面承诺】+【正文结构】+【行动价值】。它不是单纯靠标题，而是把【具体痛点】转成了【可执行方案/可保存清单/可讨论问题】。
```

## 可复制点

```text
可复制：用【标题结构】切入；封面采用【视觉结构】；正文按【步骤/清单/模块/案例】组织；结尾设置【评论问题/收藏理由/下一篇预告】。
```

Keep each analysis field concise but specific. Prefer concrete mechanisms over adjectives.
