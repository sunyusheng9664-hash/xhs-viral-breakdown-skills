# Video Analysis Templates
Use these as structure and diagnostic framework, not as boilerplate text.
Each analysis must be specific, evidence-based, and operational.

## 爆款拆解
使用「时间轴 + 信息密度 + 信任建立 + 转化设计」四维诊断模型：

### 时间轴维度
按以下段落分析视频结构。如果字幕时间戳可用，标注具体时间点；如果不可用，按相对比例估算。

```text
第一段：0-3秒（黄金钩子期）
- 钩子类型：【悬念钩子/痛点钩子/反常识钩子/权威钩子/情绪钩子/直接结果钩子】
- 具体话术/画面：【字幕显示的具体内容，或从 metadata 推断的画面描述】
- 核心任务：【在3秒内让用户产生"不划走"的决策】
- 决策机制：【用户为什么决定继续看——好奇心/恐惧感/认同感/利益期待】

第二段：3-10秒（问题建立期）
- 内容功能：【放大痛点/建立共鸣/提出反常识观点/展示权威背景】
- 信息交付：【给了用户什么新信息——一个数据/一个故事/一个误解】
- 情绪曲线：【焦虑上升/希望建立/愤怒激发/好奇加深】

第三段：10-30秒（价值交付期）
- 内容功能：【方法论讲解/案例展示/步骤拆解/对比论证】
- 信息密度：【高（每3秒一个新信息点）/中（适当停顿解释）/低（以展示为主）】
- 信任信号：【在哪里插入了数据/案例/个人经历来建立可信度】

第四段：30秒-结束（转化收尾期）
- 内容功能：【总结要点/强化记忆/CTA引导/情绪升华】
- CTA 类型：【关注/评论/收藏/私信/跳转/无明确CTA】
- 收尾强度：【强（明确指令）/中（暗示行动）/弱（自然结束）】
```

### 信息密度维度

```text
高密度段识别：
- 位置：【时间段】
- 特征：【信息量大，句子短，节奏快】
- 功能：【知识交付/步骤罗列/数据轰炸】

低密度段识别：
- 位置：【时间段】
- 特征：【停顿、过渡句、情绪铺垫、视觉展示】
- 功能：【让用户消化信息/切换情绪/建立期待】

密度变化点分析：
- 哪里加速了：【从慢到快的位置和原因——通常是进入核心方法论】
- 哪里减速了：【从快到慢的位置和原因——通常是强调重点或转换话题】
- 为什么这样设计：【加速是为了保持注意力，减速是为了强化记忆点】
```

### 信任建立维度

```text
第一层：真实感（让用户觉得"这是个真人，不是营销号"）
- 语言特征：【口语化表达/方言/口误/不完美的表达】
- 内容特征：【个人经历/失败案例/具体细节（时间/地点/金额）】
- 字幕证据：【"我当时真的傻了""说实话""我自己也踩过坑"等】

第二层：专业性（让用户觉得"这人懂行"）
- 知识密度：【专业术语使用/行业洞察/方法论命名】
- 证据类型：【数据/案例/对比/逻辑推导】
- 权威信号：【从业年限/服务客户数/成果展示/认证】

第三层：共情力（让用户觉得"这人懂我"）
- 痛点描述精度：【是否精确描述了用户的隐性痛点】
- 情绪同步：【是否在用户会焦虑的地方表达了焦虑，在用户会期待的地方给予希望】
- 身份认同：【是否使用了"我们打工人""你们做自媒体的"等身份标签】

信任层次评估：
这个视频主要依赖第【一/二/三】层信任建立。如果是第一层为主，可复制性高；如果是第二层为主，需要积累专业背书；如果是第三层为主，需要深度理解用户。
```

### 转化设计维度

```text
CTA 识别：
- CTA 类型：【硬CTA（"关注我""评论领资料"）/软CTA（"你们自己试试看""下期讲X"）/无CTA】
- CTA 位置：【结尾/中段/开头/多处插入】
- CTA 时机：【情绪高点/信息交付完毕/悬念制造后】

行动门槛评估：
- 用户需要付出什么：【注意力/时间/金钱/个人信息/社交风险】
- 门槛高低：【低（一键关注）/中（评论一句话）/高（私信+等待回复）】
- 即时回报：【用户执行CTA后能否立即获得反馈/价值】

转化链路设计：
- 第一步：【视频内引导用户做什么】
- 第二步：【用户行动后，作者如何承接（如评论区回复/私信自动回复）】
- 第三步：【最终转化目标是什么（关注/私域/产品）】
```

### 爆款拆解综合结论

```text
这条视频的核心爆款机制：
【用___钩子解决"为什么看"，用___结构解决"继续看"，用___信任信号解决"信不信"，用___设计解决"动不动（互动/转化）"。】

与同类视频的差异化：
【同类视频通常___，这条视频___，差异带来的效果是___。】

关键成功要素排序（从高到低）：
1. 【最重要的一个要素】——如果没有这个，数据会下降___%
2. 【次重要要素】
3. 【第三重要要素】
```

## 可复制部分
必须拆解为**可独立复用的模块**，每个模块含功能、结构、变量、适用场景。

### 模块提取格式

```text
模块1：【模块名称，如"反常识钩子"】
- 功能：【在3秒内制造认知冲突，让用户产生"我倒要看看"的心理】
- 结构：【"你以为X，其实Y" / "90%的人都做错了" / "为什么我放弃了X"】
- 变量替换：【X=常见做法/普遍认知；Y=反常识事实/更好方法】
- 适用场景：【教程类/认知类/工具推荐类视频】
- 时长建议：【3-5秒】
- 字幕节奏：【短句，停顿，重音在反常识关键词上】

模块2：【模块名称，如"痛点放大三段式"】
- 功能：【建立共鸣，让用户觉得"这说的就是我"】
- 结构：【"你是不是也___" → "每次___都特别___" → "其实根本原因是___"】
- 变量替换：【第一空=具体场景；第二空=负面感受；第三空=深层原因】
- 适用场景：【任何需要建立共鸣的视频开头】
- 时长建议：【5-10秒】
- 字幕节奏：【逐步加速，在"其实根本原因是"处放慢，制造期待】

模块3：【模块名称，如"方法论步骤清单"】
- 功能：【交付核心价值，让用户觉得"学到了"】
- 结构：【"第一步，___；第二步，___；第三步，___" / "只需要做X件事：1...2...3..."】
- 变量替换：【步骤内容=具体动作；数字=步骤数量】
- 适用场景：【教程类/干货类】
- 时长建议：【占总时长40-50%】
- 字幕节奏：【每步一个停顿，关键动作加粗/变色】

模块4：【模块名称，如"低门槛CTA"】
- 功能：【在不破坏体验的前提下引导互动】
- 结构：【"如果你也想___，可以先___" / "___这个方法，你们自己试试看"】
- 变量替换：【第一空=用户想要的结果；第二空=最简单的第一步行动】
- 适用场景：【任何需要引导关注的视频结尾】
- 时长建议：【3-5秒】
- 字幕节奏：【放缓，友好语气，避免命令感】
```

### 脚本公式提取

```text
完整脚本公式（可替换变量版本）：

【钩子类型】开头（3秒）
+ 【痛点放大/问题建立】（7秒）
+ 【价值交付：方法/步骤/案例】（主体时长）
+ 【信任强化：数据/个人经历/成果】（穿插或集中）
+ 【CTA设计】（3-5秒）

总时长建议：【根据平台特性和内容复杂度】
信息密度曲线：【高→低→高→低（波浪式）/ 持续高（干货型）/ 低→高→低（故事型）】
```

### 节奏模板

```text
节奏类型A：干货密集型
- 特征：语速快，信息量大，每3秒一个要点
- 适用：教程、清单、工具推荐
- 风险：容易疲劳，需要在每30秒处插入"缓冲句"
- 可复制要素：【短句+停顿+关键信息重复】

节奏类型B：故事叙述型
- 特征：语速中等，有情感起伏，信息密度不均匀
- 适用：个人经历、案例复盘、认知分享
- 关键：在"转折点"处放慢，在"背景铺垫"处加速
- 可复制要素：【"当时→然后→但是→最后"的时间线结构】

节奏类型C：观点论证型
- 特征：先快后慢，开头抛出反常识观点，中段逐步论证
- 适用：认知颠覆、行业洞察、方法论介绍
- 关键：在论证高潮处使用数据或案例"砸"用户
- 可复制要素：【"观点→质疑→证据→结论"的逻辑链】
```

## Subtitle-Limited Analysis
When only subtitles are available and frames were not inspected, follow these rules:

- Use time stamps from subtitle data to build the timeline analysis.
- For visual elements, use cautious phrasing: `字幕显示...` / `适合配合...画面` / `从口播结构看...`
- Do not claim visual details unless metadata explicitly describes them.
- Mark density changes based on subtitle text length and frequency, not visual inspection.
- For trust analysis, rely on language patterns (colloquialisms, personal pronouns, specific details).

**Required subtitle-limited disclaimer** (insert at beginning of analysis if applicable):
```text
【分析限制说明】本分析仅基于字幕文本和 metadata，未检视视频画面。涉及视觉元素的部分为推测，已标注"推测"。
```

## Analysis Quality Checklist
Before outputting final analysis, confirm:
- [ ] 时间轴维度是否覆盖了0-3秒、3-10秒、10-30秒、30秒+四个段落？
- [ ] 每段是否标注了钩子类型、信息交付、情绪/功能？
- [ ] 信息密度维度是否识别了高密度段、低密度段和变化点？
- [ ] 信任建立是否区分了三层（真实感/专业性/共情力）？
- [ ] 转化设计是否识别了CTA类型、位置、时机和行动门槛？
- [ ] 关键成功要素是否做了排序（区分核心要素和次要要素）？
- [ ] 可复制部分是否拆解为独立模块（含功能/结构/变量/适用场景）？
- [ ] 是否提取了完整脚本公式（可替换变量版本）？
- [ ] 全篇是否没有出现"节奏好""有感染力""内容优质"等无证据形容词？
- [ ] 如果仅基于字幕分析，是否插入了分析限制说明？
