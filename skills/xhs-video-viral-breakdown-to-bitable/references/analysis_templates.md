# Video Analysis Templates

Use these as structure, not boilerplate text.

## 爆款拆解

```text
这条视频用【权威/趋势/反常识/强痛点】开头，先制造【用户问题】，再通过【清单/案例/教程/对比】展开。口播结构是【钩子 -> 痛点 -> 方法 -> 结果 -> CTA】，适合爆的原因在于【收藏价值/焦虑缓解/工具可操作/结果明确】。
```

Required elements:

- opening hook
- pain point
- structure
- evidence from subtitle or metadata
- conversion/CTA design if visible

## 可复制部分

```text
可复制：用【趋势/权威/错误用法批判】开头；中段用【工具名 -> 痛点 -> 能力 -> 结果】组织；画面配【真实界面/结果截图/步骤演示】；结尾给【低门槛操作/评论领取/下期预告】。
```

Do not repeat the source video. Abstract the reusable script formula.

## Subtitle-Limited Analysis

When only subtitles are available and frames were not inspected, say so implicitly through cautious wording:

- `字幕显示...`
- `适合配合...画面`
- `从口播结构看...`

Avoid:

- `画面展示了...` unless actually inspected.

## v1.1.0 诊断式分析要求

- 爆款拆解使用四维框架：时间轴（钩子、问题建立、价值交付、收尾）、信息密度、信任建立（真实感/专业性/共情力）、转化设计（CTA 类型、位置、门槛、回报）。
- 有字幕时间戳时可标注时间；没有时只用相对阶段，不能伪造精确秒数。仅有字幕和元数据时，开头注明“本分析仅基于字幕文本和公开元数据，未检视视频画面”。
- 可复制部分拆成独立模块，每个模块写功能、结构、可替换变量、适用场景和节奏；结尾给出“钩子 → 问题建立 → 价值交付 → 信任强化 → CTA”的脚本公式。
- 无明确 CTA、未观察到画面、或无法确认的数据，明确写“未观察到”或“无法确认”；禁止用“节奏好”“有感染力”等无证据结论。
