# v1.3.1 更新说明

这是一项针对 v1.3.0 的紧急修复。v1.3.0 已经创建了“博主主页”数据表，但博主主页链接仍会被误判为图文，无法自动进入对应数据表；v1.3.1 补齐了这条链路。

## 修复内容

- 跟随小红书分享短链后，根据最终 `/user/profile/` 地址识别博主主页，不再用 `/m/` 猜测内容类型；
- 从未登录公开主页提取账号名称、小红书号、简介、头像、粉丝与获赞收藏总量，以及当前可见笔记样本；
- 新增博主分析模板，明确区分公开事实、样本分析和无法确认的数据；
- Excel 备份新增“博主主页”工作表；
- 博主记录自动写入统一 Base 的“博主主页”数据表，并上传头像；Agent 能保存主页截图时同步上传“主页截图”；
- 图文和视频会提取公开 `author_id`，仅在与博主 `user_id` 一致时填写“所属博主”；
- 重复提交同一主页时复用已有记录，并补充可确认的分析字段和附件；
- 返回已关联笔记数和未关联原因，避免把同批但不同作者的链接错误关联。

## 用户需要做什么

把仓库链接重新发给 Agent，并说明：“请更新到 v1.3.1 后，重新处理这三条小红书链接。”

如果 v1.3.0 已经完成统一多维表升级，v1.3.1 不会重新迁移或新建 Base，会继续使用现有“图文笔记”“视频笔记”“博主主页”三张数据表。

## 权限与安全边界

- 不要求登录小红书，不执行点赞、评论、关注或发布；
- 只读取用户主动提供的公开链接；
- 不通过昵称或“同批提交”猜测作者关系；
- 不填写 IP 属地、公开认证信息、账号等级或账号内部 ID；
- 未登录公开页看不到的数据留空并写入“数据限制”。

## 已验证

- 本地自动化测试覆盖主页识别、头像附件、博主 Excel、重复记录复用和所属博主关联；
- 使用本次 WorkBuddy 用户的三条实际短链回归，分别正确识别为图文、视频和博主主页；
- 三条链接的作者 ID 不一致，因此会分别归档但不会错误关联。
