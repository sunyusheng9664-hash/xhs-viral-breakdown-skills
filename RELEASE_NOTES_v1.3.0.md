# v1.3.0 更新说明

以前图文和视频分别放在两个多维表里，查找、关联博主和制作看板都要跨文件处理。v1.3.0 会把它们整理成一个“小红书内容工作台”。

## 更新后会得到什么

- 所有内容整合到同一个飞书多维表文件，更方便后期维护；
- `图文笔记`、`视频笔记`、`博主主页`三张数据表；
- 图文和视频都可以通过“所属博主”关联到博主主页；
- 原视频记录和封面附件会复制到统一工作台；
- 后续新增字段、筛选、统计和看板都可以在同一个文件中完成。

## 旧用户升级流程

把更新后的 GitHub 仓库链接交给正在使用的 Agent，并告诉它“Skill 有更新，请按仓库说明更新”。新版 Skill 首次运行时会主动：

1. 识别这是首次安装还是旧版升级；
2. 检查飞书 Base 访问权限，权限不足时先发起最小范围授权引导；
3. 展示本次迁移会改什么、需要哪些权限、明确不会做什么；
4. 等待用户明确同意后，使用一次性 `plan_id` 执行迁移；
5. 生成升级前后报告，并更新本机绑定配置。

## 需要的权限

- 读取现有多维表、记录和附件；
- 创建或改名数据表与字段；
- 复制记录、下载原附件并上传到新表；
- 更新本机 Skill 绑定配置。

不会申请小红书登录权限，不会发布、点赞或评论，也不会删除原视频库。bot 身份只通过飞书开发者后台补权限，不执行用户登录；user 身份只申请 Base 业务域的最小授权。

## 安全边界

- 原视频库保留为备份；
- 旧数据、自定义字段和自定义视图不删除；
- 未明确授权不执行迁移；
- 表结构或来源发生变化时，旧 `plan_id` 自动失效；
- 删除旧库和历史图片修复需要再次单独授权。
